# V2 终极提交文案

> 整理日期：2026-04-03
> 用途：未来正式版提交说明备用

## 提交标题

```text
feat!: 将 V2 UI 架构从 iframe SPA 迁移到原生 DOM 挂载
```

## 提交正文

```text
BREAKING CHANGE: 引入 V2 统一工作区，并完成从基于 iframe 的 SPA 承载模式到原生 DOM 挂载的 UI 架构迁移。

本次发布启动了一项全面的 UI 架构迁移工作，将发布工具从基于 iframe 的 SPA 系统重构为原生挂载的统一工作区，并以飞书/字节系信息架构原则和思源原生 UI 优先策略为指导。

架构转变：
- 以单一统一工作区外壳取代历史上的 iframe + SPA 双系统
- 从基于页面的跳转导航转向同一工作区内的渐进式展开
- 将真实 DOM 挂载确立为所有 V2 能力的唯一基础
- 废弃以 iframe 页面承载所有新 V2 功能的方式

统一工作区模型：
- 一个同时承载快速发布主视图和设置展开态的统一外壳
- 由品牌区、导航区和内容工作区组成的整体布局
- 渐进式展开路径：主视图 -> 设置导航 -> 平台选择 -> 配置表单
- 在所有用户流程中保持一致的心智模型

迁移策略：
- 引入 `useV2UI` 功能开关，以支持渐进式迁移并保留完整回退能力
- 在迁移期间保留旧版 UI 作为完整可用的回退路径
- 保持现有配置格式不变，无需进行数据迁移
- 复用既有业务逻辑，仅重构表现层

里程碑框架（M0-M6）：
- M0：建立入口治理、统一偏好配置读取和 V2 Host
- M1：建立样式隔离，并以思源原生 UI 作为首选样式基线
- M2-M3：交付快速发布主视图与完整发布闭环
- M4：扩展设置能力，纳入账号列表、图床设置和平台配置桥接
- M5：逐步替换剩余的旧版设置能力
- M6：完成最终收敛规划，并定义 iframe 退役路线图

设计原则：
- 优先使用思源原生 UI，仅在必要时补充自定义样式
- 对用户和开发者都保持高度确定性
- 通过渐进式展开呈现复杂度，而不是一次性全部暴露
- 配置与状态保持单一事实来源
- 在每个阶段都保留可靠的故障恢复路径

影响范围：
- 入口层，包括顶栏入口和文档菜单
- V2 Host 挂载层
- `UnifiedWorkspaceShell` 组件架构
- 快速发布主界面
- 设置展开态的状态管理
- 图床设置和偏好设置
- 平台配置桥接能力
- 构建链路，包括用于独立验证 V2 的 `vite.v2.config.ts`

迁移指南：
- 在偏好设置中启用“使用新版 UI（实验性）”即可切换到 V2
- 所有平台适配器无需修改即可继续工作
- 某些自定义的界面级设置可能需要在新 UI 中重新配置
- 可随时通过偏好设置中的开关回退到旧版路径
```
