# 图片上传错误可查看修复

## 问题根因

`baseExtendApi.handlePictures()` 的 Bundled 分支在图片上传失败时:
1. catch块只 `logger.error` + `pushMsg`(推到思源消息栏)
2. **不抛出异常**，不附加错误到post上
3. 导致 `doSinglePublish` 返回 `status: true`，图片错误完全丢失
4. V2App 显示"更新成功"绿色卡片，图片错误信息不可见

## 修复方案

### Task 1: Post模型添加图片错误字段

**文件**: `zhi-blog-api` 中的 Post 类型（外部依赖，不可直接改）

**替代方案**: 在 `doSinglePublish` 层面捕获，不修改Post模型。在 `usePublish.ts` 的 `doSinglePublish` 中用 `try/catch` 包裹 `api.preEditPost()`，如果 preEditPost 抛出图片异常就捕获。

**最终方案**: 修改 `handlePictures` 让图片上传错误不再被静默吞掉，改为在 post 上设置自定义属性传递错误，然后在 `doSinglePublish` 检查该属性。

**文件**: `src/adaptors/base/baseExtendApi.ts`

- L535-551: 修改catch块，不再吞掉异常
  - 收集图片上传错误到 `imageUploadErrors` 数组（挂在 post 自定义属性上）
  - 将原始错误信息 `e.message` 保留下来
  - 删除"请打开开发者工具查看错误日志"的 pushMsg，改为在post上附带错误

```
// 改前
} catch (e) {
  const message = e.message || e
  let ignoreError = false
  if (message.includes(BaseError.NO_PAGE_ID_FOUND_IN_MEDIA_MACRO_MODE)) {
    ignoreError = true
  }
  if (!ignoreError) {
    const errMsg2 = "文章可能已经发布成功，但是平台图片上传失败。请打开「开发者工具」查看错误日志"
    this.logger.error(errMsg2, e)
    await this.kernelApi.pushMsg({ msg: errMsg2, timeout: 7000 })
  }
}

// 改后
} catch (e) {
  const message = e.message || e
  let ignoreError = false
  if (message.includes(BaseError.NO_PAGE_ID_FOUND_IN_MEDIA_MACRO_MODE)) {
    ignoreError = true
  }
  if (!ignoreError) {
    const imgErrMsg = `图片上传失败: ${message}`
    this.logger.error(imgErrMsg, e)
    if (!post.imageUploadErrors) {
      (post as any).imageUploadErrors = []
    }
    (post as any).imageUploadErrors.push(imgErrMsg)
  }
}
```

### Task 2: doSinglePublish传播图片错误

**文件**: `src/composables/usePublish.ts`

在 `doSinglePublish` 中，`api.preEditPost()` 之后检查 `finalPost.imageUploadErrors`，将其合并到返回的 errMsg 中:

```typescript
// L113 之后 (const finalPost = await api.preEditPost(...))
const imageErrors = (finalPost as any).imageUploadErrors as string[] | undefined
if (imageErrors && imageErrors.length > 0) {
  singleFormData.errMsg = imageErrors.join("\n")
}
```

关键: **不改变 status** — 文章确实发布成功了，但 errMsg 中包含图片警告。这样返回对象 `{ status: true, errMsg: "图片上传失败: xxx" }` 表示"部分成功"。
