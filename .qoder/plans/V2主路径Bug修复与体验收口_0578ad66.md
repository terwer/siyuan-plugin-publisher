# V2 主路径 i18n Bug 修复

## 必做：缺失 i18n key 导致初始化错误显示原始 key

**文件**: `siyuan/i18n/zh_CN.json` + `siyuan/i18n/en_US.json`

V2App.vue L56 使用 `t("v2.quickPublish.error.initFailed")`，L58 使用 `t("v2.common.retry")`，但这两个 key 在两个 i18n 文件中均不存在。

当 V2 初始化失败时，用户看到的不是"初始化失败"和"重试"，而是原始 key 字符串。

**修复**: 在两个 i18n 文件中添加：
```
// zh_CN
"v2.quickPublish.error.initFailed": "初始化失败",
"v2.common.retry": "重试",

// en_US
"v2.quickPublish.error.initFailed": "Initialization failed",
"v2.common.retry": "Retry",
```

## 不纳入本次的议题

- **V2App.vue v-else 兜底**：在当前 useV2Settings 受控状态模型下无证据证明会触发，如需处理应作为防御性兜底治理，不是主路径 bug
- **发布状态残留**：本质是状态区语义与卡片关系未定义清楚，属于交互策略问题，需要先定义 active card / 选中态概念，不是现成可复现 bug

## 验证

- `pnpm build:v2` 通过
- V2 初始化失败时显示"初始化失败"而非原始 key
