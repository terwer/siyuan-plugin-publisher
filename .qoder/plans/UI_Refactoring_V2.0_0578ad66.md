# 发布工具 UI 重构 V2.0 计划

## 核心设计理念（参考飞书/字节设计）

### 主界面优先原则
- **用户第一眼看到的是已配置的平台列表**，点击即可发布
- **设置是次要入口**，以小按钮形式存在（如右上角齿轮图标）
- **设计图展示的是"展开设置后"的状态**，不是主界面

### 渐进式迁移策略（重要）
- **新旧 UI 完全共存**：新 UI 代码完全隔离，旧 UI 保持不动
- **功能开关控制**：通过配置切换新旧 UI，可随时回退
- **新 UI 测试通过后再逐步替换**：至少经过几个版本验证
- **禁止丢失任何功能**：仅做 UI 升级，功能逻辑复用现有后端

### 前后端隔离原则
- **新 UI 前端代码完全隔离**：`src/components/v2/` 目录独立
- **后端逻辑尽可能复用**：复用 `src/adaptors/`、`siyuan/store/` 等现有逻辑
- **配置格式保持不变**：确保新旧 UI 读写同一配置

---

## 项目概述
基于 Figma 设计图 V2.0 对发布工具插件进行完全重构：
- 将 iframe 弹窗改为思源笔记原生插件弹窗 UI
- 主界面为"快速发布面板"，设置作为二级入口
- 根级别 CSS 隔离，避免与思源笔记样式冲突

## 设计分析

### 主界面布局（快速发布面板）
- **顶部栏**：标题"发布到" + 右上角设置按钮（小齿轮图标）
- **平台网格/列表**：已配置的平台以卡片形式展示
  - 每个卡片：平台图标 + 平台名称 + 发布按钮
  - 点击直接发布当前文档到对应平台
- **空状态**：未配置平台时显示"添加平台"引导

### 设置界面布局（点击设置按钮后展开）
- **左侧导航栏**：账号设置、图床设置、偏好设置
- **中间内容区**：根据选择显示对应列表
- **右侧详情区**：配置表单（参考 Figma 设计图）

### 用户操作流程（新版）

**快速发布流程：**
1. 用户打开插件，看到已配置的平台列表
2. 点击任意平台卡片，直接发布当前文档
3. 显示发布进度和结果

**添加账号流程：**
1. 用户点击设置按钮（齿轮图标）
2. 进入设置界面，显示"添加账号"按钮
3. 显示平台分类列表（通用平台、GitHub、GitLab 等）
4. 选择平台后显示配置表单
5. 保存后返回主界面，新平台出现在列表中

### 视觉设计规范
- 主色调：科技蓝 (#4080FF)
- 背景边框：中性灰色系
- 圆角：卡片和按钮使用大圆角（8-12px）
- 间距：基于 8px 的间距系统
- 风格：简洁专业的 SaaS B 端风格（参考飞书）

## 关键技术决策

1. **CSS 隔离**：使用 `.syp-v2` 命名空间，完全隔离新 UI 样式
2. **代码隔离**：新 UI 代码完全放在 `src/components/v2/` 目录，不动现有代码
3. **后端复用**：完全复用 `adaptors/`、`stores/`、`composables/` 现有逻辑
4. **功能开关**：通过偏好设置控制新旧 UI 切换，可随时回退
5. **渐进式迁移**：新旧 UI 共存至少 3 个版本，稳定后再考虑移除旧代码
6. **主界面优先**：用户第一眼看到的是快速发布面板，设置是次要入口
7. **飞书风格**：参考飞书/字节的设计，简洁专业，主操作突出
