## 项目结构
该仓库采用"前端应用 + 多目标构建脚本 + 配置文件"的组织方式：
- 前端应用位于 src 目录，使用 Vite + Vue 3 + TypeScript
- **新增** UI V2 架构位于 src/v2 目录，包含新的组件结构和样式系统
- 构建脚本集中于 scripts 目录，按目标拆分 Python 脚本
- 配置文件包括 Vite、TypeScript、ESLint、Prettier 等
- **新增** Vite V2 配置文件专门用于 UI V2 构建
- 产物目标包括：Siyuan 插件包、挂件包、浏览器扩展包、Nginx 部署包、**新增** V2 版本构建产物

### Vite V2 配置
- **新增** 独立构建配置
  - 专门用于 UI V2 构建的独立配置文件，使用 dist-v2 作为输出目录。
  - 采用库模式构建（formats: ["cjs"]），入口为 siyuan/index.ts。
- **新增** 实时开发支持
  - 支持 watch 模式下的实时热更新，使用 livereload 插件。
  - 监听静态资源文件（plugin.json、README*.md、LICENSE、icon.png、preview.png、siyuan/i18n/*.json）。
- **新增** 静态资源处理
  - 自动复制静态资源到 dist-v2 目录，包括配置文件、国际化文件等。
  - 支持复杂的文件复制逻辑和目录创建。
- **新增** Node polyfill
  - 配置 nodePolyfills 插件，支持 Buffer、global、process 等全局对象。
- **新增** 构建优化
  - CSS 代码分割关闭，统一输出到 index.css。
  - 自定义文件命名策略，CSS 输出到 index.css，其他资源输出到 assets/[name].[ext]。

## 故障排查指南
- 构建失败
  - 检查 BUILD_TYPE 是否正确设置；确认目标目录权限与磁盘空间。
  - 查看各目标脚本的环境变量设置与命令拼接是否正确。
- 产物异常
  - 若挂件/扩展缺少资源，确认脚本已复制 icon、README、manifest 等文件。
  - 若缓存导致页面不更新，确认 HTML 注入的时间戳参数是否生效。
- 版本不一致
  - 确认 package.json 的 version 与各目标脚本的打包命名一致。
- **新增** V2 版本问题
  - 检查 vite.v2.config.ts 配置是否正确，特别是输出目录和入口文件。
  - 确认静态资源复制插件是否正常工作，检查 dist-v2 目录结构。
  - 验证 UI V2 组件的导入路径和依赖是否正确。
