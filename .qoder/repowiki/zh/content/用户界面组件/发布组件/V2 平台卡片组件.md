## 简介

V2 平台卡片组件是思源笔记发布工具插件中的核心 UI 组件，负责展示支持的发布平台信息。该组件提供了直观的卡片式界面，显示平台图标、名称、授权状态和发布状态，为用户提供快速发布功能。

**更新** 该组件现已大幅增强，新增了预览功能、删除确认对话框和危险警告系统，为用户提供了更加安全和便捷的操作体验。组件支持预览已发布内容、确认删除操作以及直观的危险操作提示。

该组件采用现代化的设计理念，具有响应式布局、优雅的视觉效果和良好的用户体验。组件支持多种平台类型，包括通用平台、GitHub/GitLab 生统、自定义平台等，并提供了完整的状态管理和数据绑定机制。

## 核心组件

### V2PlatformCard 组件

V2PlatformCard 是平台卡片的核心组件，负责渲染单个平台的信息卡片。该组件具有以下特性：

- **响应式设计**：支持不同屏幕尺寸的自适应布局
- **增强的状态指示**：通过 `is-ready` 和 `is-disabled` 类提供清晰的视觉状态反馈
- **图标支持**：支持 SVG 和图片格式的平台图标
- **无障碍访问**：提供适当的 ARIA 属性和标题提示，包括 `aria-disabled` 属性
- **动态样式绑定**：根据授权状态动态应用 CSS 类
- **新增预览功能**：支持查看已发布文章的预览链接
- **新增删除确认**：删除操作前的安全确认对话框
- **新增危险警告**：删除按钮的危险样式和警告提示

组件的主要属性包括：
- `platformName`：平台显示名称
- `platformIcon`：平台图标（SVG 代码或图片路径）
- `isAuthorized`：授权状态标志
- `isPublished`：发布状态标志
- `tooltipText`：悬停提示文本
- `isProcessing`：处理状态标志
- `previewLink`：预览链接地址
- `isFailed`：失败状态标志

**更新** 组件现在支持三种主要状态类：
- `is-ready`：当平台已授权时应用，显示绿色状态指示
- `is-disabled`：当平台未授权时应用，显示半透明效果和红色状态指示
- `is-danger`：当操作失败时应用，显示危险状态样式

## 性能考虑

### 渲染优化

V2 平台卡片组件在设计时充分考虑了性能优化：

1. **虚拟 DOM 优化**：使用 Vue 3 的 Composition API 提供更好的渲染性能
2. **条件渲染**：通过 `v-if` 和 `v-show` 控制组件的显示和隐藏
3. **懒加载**：平台图标采用延迟加载策略
4. **内存管理**：合理使用响应式数据，避免内存泄漏
5. **状态类缓存**：CSS 类的切换是纯样式操作，性能开销极小
6. **新增优化**：确认对话框采用条件渲染，仅在需要时创建

### 数据缓存策略

组件采用了多层数据缓存机制：

- **设置缓存**：发布设置存储在本地，减少网络请求
- **状态缓存**：平台状态在组件实例中缓存
- **计算属性**：使用 `computed` 属性避免重复计算
- **状态类计算**：授权状态的计算结果会被缓存
- **新增缓存**：预览链接和确认状态进行缓存管理

### 样式性能

- **CSS 作用域**：使用 scoped CSS 避免样式冲突
- **样式复用**：共享样式变量和基础样式
- **媒体查询**：响应式设计减少不必要的重排
- **状态类切换**：纯 CSS 动画，无 JavaScript 计算开销
- **新增样式**：危险警告样式采用高效的 CSS 实现
